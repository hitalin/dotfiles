# コンテキストウィンドウ管理

## 基本方針

コンテキストは有限リソース。無駄な消費を避け、重要な情報を維持する。

## ルール

- 調査・探索はサブエージェント（Explore）に委任し、メインコンテキストを保護
- コンテキスト使用率 70% 超えで `/compact <要約指示>` を実行
- 大きなファイルは必要な範囲のみ読む（offset/limit 指定）
- 巨大な出力が予想されるコマンドは `head` や `tail` で制限

## Document & Clear パターン

大きなタスクは以下のサイクルで進める：

1. 調査・実装を行う
2. 成果をファイルに書き出す（コード、メモ）
3. `/compact` で不要なコンテキストを圧縮
4. 次のステップに進む
