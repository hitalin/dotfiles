# コーディングスタイル

## 一般原則

- YAGNI: 必要になるまで実装しない
- KISS: シンプルに保つ
- DRY: ただし早すぎる抽象化は避ける（3回繰り返すまで待つ）

## 命名規則

- 変数・関数: 英語で、その役割を明確に表す名前
- 略語は一般的なもののみ（URL, ID, HTTP等）
- コメントは日本語可

## エラーハンドリング

- システム境界でのみ検証（ユーザー入力、外部API）
- 内部コードは信頼する
- 起こり得ないシナリオのエラーハンドリングは不要

## テスト

- テストファーストを推奨
- テスト名は「何を」「どうすると」「どうなる」を表す
- モックは最小限に
